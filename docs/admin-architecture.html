Портфолио — админка персонализации

Сабдомен-админка + нейронка кастомят портфолио под каждую компанию. Публичный сайт уже читает компанию из ?to=slug. Это наметки архитектуры — наведись на варианты, выбери развилки, скопируй JSON в конце.

1. Что кастомизируем

Один публичный сайт → много «версий под компанию»

Открываешь админку, вставляешь вакансию, нейронка собирает релевантную версию, ты правишь и публикуешь. Рекрутёр открывает site/?to=plata — видит подобранное под него.

2. Поток: вакансия → версия

Нейронка не публикует сама. Она готовит черновик из заранее заготовленного материала (кейсы, текстовые блоки, опыт). Ты — редактор-гейт. Это снимает риск, что AI напишет ерунду на живой сайт.

5. Модель данных (черновик)

Грубый набросок, чтобы было от чего отталкиваться — поля уточним после развилок выше.

Company {
  slug            // "plata"  → ?to=plata
  name            // "Plata"  → "Hello Plata"
  vacancyText     // вставленный текст вакансии (вход для AI)
  caseIds[3]      // выбранные кейсы из реестра, по порядку
  emphasis[]      // id фиксированных фраз, которые подчёркиваем
  textOverrides?  // опционально: правки блоков под компанию
  expHighlights[] // id мест опыта для подсветки
  video { url, duration } // кастом или дефолт
  updatedAt
}

CaseEntry {        // реестр кейсов (страница остаётся React-компонентом)
  id              // "zinda"
  title, year, category
  tags[]          // "финтех", "b2b", "mobile"
  matchKeywords[] // под матчинг вакансии
  component       // какой компонент рендерить
}

TextBlock {        // центральный текст — авторим мы
  id, text
  emphasizable[]  // какие фразы AI может подчёркивать
}

ExperienceItem { id, company, role, period, tags[] }
